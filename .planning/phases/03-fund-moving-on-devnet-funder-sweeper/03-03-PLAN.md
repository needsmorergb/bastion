<objective>
Add the D-10 retire guard so a swept session can be retired safely without orphaning token value.

Delivers SESS-07 (retire a session keystore after it has been swept) with the guard that refuses hard-delete when a nonzero token balance remains — necessary because v1 sweeps SOL-only and leaves nonzero-token ATAs untouched (D-06), so deleting such a keystore would orphan real token value.

Purpose: keep `session.retire()` synchronous and RPC-transport-agnostic (it stays the module that owns the keystore file, gaining no httpx dependency) while enforcing the guard from a caller-supplied, freshly-read token-account list.

Output: a new `KeystoreRetireError`, the extended `retire()` signature + guard, and unit tests.
</objective>
