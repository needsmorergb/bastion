<objective>
Create `keystore/vault.py` with an isolated `load_vault()` and lock in the
structural vault/session split with an AST-based import-isolation test. This is
the load-bearing precondition for SEC-02 (Phase 3: sweeps need no vault secret)
and SEC-03 (Phase 5: scoring/LLM cannot reach the keystore) — established now,
before any fund-moving code exists, so those later invariants are provable
rather than hoped-for.

Purpose: The vault secret is the highest-privilege key in the system. Confining
its loader to a single module that only the funder (Phase 3) may import turns a
convention into a checkable, enforced fact — Python's own import graph.

Output: `bastion/keystore/vault.py` and the static isolation test.
</objective>

| Boundary | Description |
|----------|-------------|
| env VAULT_SECRET → process | The highest-privilege key enters the process only through load_vault. |
| module import graph | Any module that can import vault.py can reach the vault secret loader — the boundary is which files import it. |
