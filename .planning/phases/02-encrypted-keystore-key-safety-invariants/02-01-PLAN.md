<objective>
Lay the Phase 2 foundation: add the two locked cryptographic dependencies
(`cryptography` for scrypt+Fernet, `solders` for Keypair) with a one-time
supply-chain sanity checkpoint, create the `bastion/keystore/` package, and
define the typed `KeystoreError` hierarchy that every other Phase 2 module
raises. This is the Wave 1 gate — nothing else in the phase can run until the
deps are pinned and the error contract exists.

Purpose: The keystore is the single highest-value target in the system. Both
new packages were flagged `SUS` by the automated legitimacy seam and manually
overridden to OK (see 02-RESEARCH.md `## Package Legitimacy Audit`); research
mandates one lightweight human confirmation before the first `uv add`. The
error hierarchy is defined first so downstream modules (crypto/session/
cloudsync/passphrase/vault) build against a stable contract, not a scavenger hunt.

Output: Two hash-pinned dependencies in `uv.lock`, an importable
`bastion/keystore/` package, and `bastion/keystore/errors.py`.
</objective>

## Artifacts this phase produces (full phase map — reference for all Phase 2 plans)

Modules (`bastion/keystore/`):
- `errors.py` — `KeystoreError`, `KeystoreWrongPassphraseError`, `KeystoreCloudSyncError`, `KeystoreConfigError` (this plan)
- `crypto.py` — `KEYSTORE_VERSION`, `KDF_N=131072`, `KDF_R=8`, `KDF_P=1`, `SALT_BYTES=16`, `encrypt_secret(passphrase, plaintext) -> dict`, `decrypt_secret(passphrase, blob) -> bytes`, `_derive_fernet_key(...)`, `_validate_kdf_params(n, r, p)` (02-02)
- `cloudsync.py` — `CLOUD_SYNC_SEGMENTS`, `detect_cloud_sync(path) -> str | None`, `check_keystore_dir(path, allow_cloud_sync=False) -> None` (02-04)
- `passphrase.py` — `MIN_PASSPHRASE_WARN_LEN`, `prompt_new_passphrase(confirm_attempts=3) -> str` (02-04)
- `session.py` — `SessionKeypair` (redacted repr), `generate() -> SessionKeypair`, `save(session, keystore_dir, passphrase, allow_cloud_sync=False) -> str`, `load(pubkey, keystore_dir, passphrase) -> SessionKeypair`, `retire(session_or_pubkey, keystore_dir) -> None` (02-05)
- `vault.py` — `load_vault(config) -> Keypair` (import-isolated) (02-03)

Keystore JSON on-disk fields (`<pubkey>.json`): `version`, `kdf` (="scrypt"), `n`, `r`, `p`, `salt` (base64), `ciphertext` (base64).

Config additions: none — Phase 2 reads `Config.keystore_dir` and reuses `config.get_passphrase()`; the cloud-sync override is a function parameter (`allow_cloud_sync`), with CLI-flag/Config wiring deferred to Phase 7.

Test files (`tests/unit/`): `test_keystore_crypto.py`, `test_keystore_session.py`, `test_keystore_cloud_sync.py`, `test_keystore_passphrase_ux.py`, `test_keystore_no_secret_leak.py`, `test_keystore_vault_isolation.py`.
