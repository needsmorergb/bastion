    - generate() returns a SessionKeypair; two calls yield distinct pubkeys (unique).
    - SessionKeypair repr/str contain "REDACTED" and never the secret bytes.
    - save() creates <keystore_dir>/<pubkey>.json; on POSIX (os.name == "posix") its mode is exactly 0o600; the file is written atomically (a partial temp is never left as the final name).
    - The saved file's parsed JSON has the versioned ciphertext-only fields and no plaintext key.

    - save() into a cloud-sync dir raises KeystoreCloudSyncError by default (delegates to check_keystore_dir); into an empty dir raises KeystoreConfigError.
    - retire() removes the keystore file and zeroizes the in-memory _secret bytearray (post-retire the bytearray is all-zero); retire tolerates an already-absent file.
  </behavior>
  <action>
    Define `SessionKeypair` as a dataclass holding `pubkey: str` and
    `_secret: bytearray` (declared `field(repr=False)`), with `__repr__`/`__str__`
    returning `f"SessionKeypair(pubkey={self.pubkey!r}, secret=REDACTED)"`
    (mirrors config.py's repr-safety), and a `zeroize()` that overwrites
    `_secret[:] = b"\\x00" * len(_secret)` (best-effort — document in the docstring
    that Python cannot guarantee wiping all copies, per Pitfall 5).

    `generate()`: create `solders.keypair.Keypair()`, wrap `bytearray(bytes(kp))`
    (64 bytes) as `_secret` and `str(kp.pubkey())` as `pubkey`.

    `save(session, keystore_dir, passphrase, allow_cloud_sync=False)`: call
    `check_keystore_dir(keystore_dir, allow_cloud_sync)` FIRST (empty-dir and
    cloud-sync guards). Build the blob via `crypto.encrypt_secret(passphrase, bytes(session._secret))`,
    `json.dumps` it, and write atomically to `<keystore_dir>/<pubkey>.json` using
    the temp-file + `os.chmod(tmp, 0o600)` + `os.replace(tmp, final)` idiom
    (Pattern 4). Note in a comment that 0600 is POSIX-real / Windows-best-effort
    (Pitfall 1). Return the final path. Never log the passphrase or secret.

    `load(pubkey, keystore_dir, passphrase)`: read `<keystore_dir>/<pubkey>.json`,
    `json.loads`, `crypto.decrypt_secret(passphrase, blob)` (propagates
    KeystoreWrongPassphraseError — do NOT catch-and-return), then
    `Keypair.from_bytes(plaintext)` as the second fail-closed check, and return a
    fresh `SessionKeypair`. The decrypted key is surfaced per-call only — never
    cached in a module global.

    `retire(session_or_pubkey, keystore_dir)`: resolve the pubkey, delete
    `<pubkey>.json` if present (best-effort, tolerate FileNotFoundError), and if a
    SessionKeypair was passed, call its `zeroize()`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-memory secret → disk | The decrypted 64-byte keypair crosses to disk only as Fernet ciphertext. |
| in-memory secret → stdout/stderr/logs | Any repr/print/log of the wrapper or an error could leak the key. |
| keystore file ← untrusted read | A wrong passphrase against a stored file must fail closed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `save()` on-disk file | critical | mitigate | Writes only the versioned ciphertext-only blob; atomic temp+`os.replace` avoids a briefly world-readable partial file (SESS-04). |
| T-02-04 | Information Disclosure | keystore file permissions | high | mitigate | `os.chmod(tmp, 0o600)` before replace; POSIX test asserts exact 0o600; Windows limitation documented + asserted best-effort (not silently claimed). Residual Windows-ACL gap accepted for a personal single-machine tool. |
| T-02-03 | Tampering / Information Disclosure | `load()` wrong passphrase | high | mitigate | Propagates KeystoreWrongPassphraseError from decrypt; `Keypair.from_bytes` second validation; never returns garbage (SESS-05). |
| T-02-02 | Information Disclosure | SessionKeypair repr / logs / errors | high | mitigate | Redacted `__repr__`/`__str__`; per-call decrypt never cached; capfd+caplog regression over the full flow asserts zero secret-shaped strings (SEC-01). |
</threat_model>
