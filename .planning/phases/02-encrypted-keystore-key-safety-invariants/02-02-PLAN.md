<objective>
Build `keystore/crypto.py` — the scrypt→Fernet cryptographic core. It derives a
Fernet key from a passphrase + per-file salt using the CONTEXT-locked scrypt
parameters (n=2¹⁷, r=8, p=1), encrypts/decrypts raw keypair bytes, serializes
to the locked versioned JSON blob (ciphertext-only on disk), and fails closed on
a wrong passphrase or malformed KDF params.

Purpose: This is the fund-safety primitive the whole keystore rests on. Getting
the fail-closed contract (SESS-05 precondition) and the ciphertext-only on-disk
format (SESS-04, SEC-01) right here means `session.py` (02-05) is a thin, safe
wrapper.

Output: `bastion/keystore/crypto.py` and its unit tests.
</objective>

  <behavior>
    - Roundtrip: encrypt_secret(pass, plaintext_bytes) → blob; decrypt_secret(pass, blob) == plaintext_bytes exactly, for a 64-byte payload.
    - Fresh salt per call: two encrypt_secret calls on the same plaintext produce different salt and different ciphertext.
    - Wrong passphrase fails closed: decrypt_secret with a different passphrase raises KeystoreWrongPassphraseError (from cryptography's InvalidToken), never returns bytes.
    - Tampered ciphertext fails closed: flipping a ciphertext byte raises KeystoreWrongPassphraseError (Fernet HMAC).
  </behavior>
  <action>
    Implement `_derive_fernet_key(passphrase, salt, n, r, p)`: construct a FRESH
    `Scrypt(salt=salt, length=32, n=n, r=r, p=p)` every call (never reuse a KDF
    instance — Pitfall 3 AlreadyFinalized), call `.derive(passphrase.encode("utf-8"))`,
    and return `base64.urlsafe_b64encode(derived)` (Fernet needs the urlsafe-b64
    form, NOT raw scrypt output — Pitfall 2). Do not pass a `backend=` argument.

    Implement `encrypt_secret(passphrase, plaintext) -> dict`: generate
    `salt = os.urandom(SALT_BYTES)`, derive the key, `Fernet(key).encrypt(plaintext)`,
    and return the versioned blob dict with base64-encoded `salt` and `ciphertext`
    plus `version`, `kdf`, `n`, `r`, `p` (locked constants). The returned dict
    contains NO plaintext-key field — the secret exists only inside `ciphertext`.

    Implement `decrypt_secret(passphrase, blob) -> bytes`: re-derive the key from
    the blob's stored `salt`/`n`/`r`/`p`, call `Fernet(key).decrypt(...)`, and
    return the plaintext. Wrap `cryptography.fernet.InvalidToken` in
    `KeystoreWrongPassphraseError` (message-only, no secret bytes) via `raise ... from exc`
    — NEVER catch-and-return None/empty. Do not log the passphrase, key, or plaintext.

    Write `tests/unit/test_keystore_crypto.py` covering the four behaviors above.
    Use a 64-byte `os.urandom(64)` payload as the stand-in keypair blob.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| passphrase → derived key | User passphrase crosses into key material via scrypt; must never be logged. |
| keystore file → decrypt | An on-disk (possibly hand-edited/corrupt) blob crosses back into the process; params and ciphertext are untrusted input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Information Disclosure | on-disk blob format | critical | mitigate | Blob is ciphertext-only (secret lives solely inside Fernet ciphertext); no plaintext-key field; unit test asserts the property (SESS-04, SEC-01). |
| T-02-03 | Tampering / Information Disclosure | `decrypt_secret` | high | mitigate | Wrong passphrase or tampered ciphertext raises KeystoreWrongPassphraseError via Fernet HMAC (InvalidToken) — never returns garbage; tested (SESS-05 precondition). |
| T-02-02 | Information Disclosure | exception messages / logs | high | mitigate | Errors are message-only; passphrase/key/plaintext never interpolated or logged. |
| T-02-08 | Tampering | stored KDF params | medium | mitigate | `_validate_kdf_params` rejects non-power-of-two `n` and non-positive `r`/`p` with KeystoreConfigError before deriving (V5 input validation). |
</threat_model>
