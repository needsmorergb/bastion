<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment (`.env` / OS env) → Local Process (config load) | Secret material (VAULT_SECRET, KEYSTORE_PASSPHRASE, TELEGRAM/PUSHOVER tokens) and safety-rail values enter here |
| Local Process → log/terminal output | Any repr/log of Config could leak a secret to disk/history |
| Local Process (config) → External Service endpoint URL | A tampered SOLANA_RPC/SOLANA_WS points transport at an attacker endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-03 | Information Disclosure | `Config` repr/str, getpass value | high | mitigate | Secret fields declared non-repr; `get_passphrase()` uses non-echoing getpass and is never logged; test asserts repr/str exclude secret values |
| T-01-04 | Tampering / Elevation | numeric safety-rail parsing (MAX_SESSION_CAP etc.) | medium | mitigate | Guarded coercion raises `ConfigError` on malformed input (V5) — a broken cap never silently becomes a permissive default; test `test_malformed_rail_fails_loudly` |
| T-01-05 | Tampering (MITM) | SOLANA_RPC / SOLANA_WS scheme | medium | mitigate | Warn on non-https/non-wss endpoint at load (loud-warning posture); test `test_non_secure_endpoint_warns`. Not a hard fail — localhost/devnet http is legitimate |
| T-01-06 | Information Disclosure | KEYSTORE_PASSPHRASE required in env | medium | mitigate | Passphrase is never required in env (getpass fallback, D-02); avoids shell-history/committed-`.env` leakage of the passphrase specifically |
</threat_model>
